# Hierarchical Clustering Explained Intuitively

## Watching Machines Build Relationships Inside Data

Welcome to the Hierarchical Clustering project in the GraphX Labs ML Concepts Portfolio.

This project is built around one central idea:

> What if clustering was not about forcing groups... but about discovering relationships step-by-step?

Hierarchical Clustering is an unsupervised learning method that builds a tree of relationships. Instead of immediately assigning points into fixed groups, it starts with individual data points and gradually merges the most similar ones.

The result is not just clusters. It is a history of how those clusters formed.

---

# Business Problem

Customer segmentation helps businesses understand how different customers behave.

The deeper question:

> Can we discover not only customer groups, but also the relationships between those groups?

This matters for marketing, personalization, recommendation systems, customer experience, and behavioral analysis.

# Clustering Intuition

Clustering groups similar data points together.

Hierarchical Clustering goes one step deeper: it shows how small relationships merge into larger relationships.

Think of:

- family trees
- friend groups forming
- species evolution trees
- folders inside folders

This makes Hierarchical Clustering especially interpretable.

---

# Dendrogram Explanation

A dendrogram is the family tree of the data.

At the bottom, each customer starts alone. As we move upward, customers merge into pairs, pairs merge into groups, and groups merge into larger branches.

The vertical height of each merge shows how far apart the groups were when they joined.

To choose clusters, we cut the dendrogram at a chosen height.

---

# Linkage Methods

Linkage methods decide how distance is measured between groups:

- **Single linkage:** closest pair between groups
- **Complete linkage:** farthest pair between groups
- **Average linkage:** average distance between group members
- **Ward linkage:** merge that keeps clusters compact

Different linkage methods can create different cluster stories.

---

# Customer Segmentation

The final model uses Ward linkage and 5 clusters to create interpretable customer groups from income and spending behavior.

Segments include:

- Premium high-value customers
- High-income cautious spenders
- Budget enthusiastic shoppers
- Low-income careful shoppers
- Balanced everyday customers

# Comparison With KMeans

KMeans is fast and centroid-based.

Hierarchical Clustering is relationship-based and explainable.

KMeans asks:

> Where are the centers?

Hierarchical Clustering asks:

> How are the points related?
