# Evaluation Metrics Explained Intuitively

## The Science of Knowing Whether Your Model Is Actually Useful

Welcome to the Evaluation Metrics project in the GraphX Labs ML Concepts Portfolio.

This project is built around one central idea:

> A model can look accurate and still completely fail in the real world.

Evaluation metrics are not just formulas. They are how machine learning teams translate model behavior into business consequences. They tell us which mistakes the model makes, how often those mistakes happen, and whether those mistakes are acceptable for the problem.

---

# Business Problem

This project uses customer churn prediction.

The model predicts whether a telecom customer is likely to leave.

The evaluation question:

> Is the model useful for retention decisions, or does it only look good on a scoreboard?

False positives and false negatives both matter:

- False positive: the model flags a loyal customer as churn risk, wasting retention budget.
- False negative: the model misses a customer who actually churns, causing preventable revenue loss.

The right metric depends on which mistake the business fears more.

# Why Evaluation Matters

Accuracy can hide failure.

In an imbalanced problem, a model can predict the majority class all the time and still look accurate. This project opens with the accuracy trap: a fraud-style example where a model achieves 99% accuracy while catching zero fraud.

The lesson:

> A metric is useful only if it matches the decision you are trying to make.

---

# Confusion Matrix Intuition

The confusion matrix shows the four ways a model can be right or wrong:

- True Negative
- True Positive
- False Positive
- False Negative

It is the foundation for accuracy, precision, recall, and F1-score.

---

# Precision vs Recall

Precision asks:

> When the model predicts churn, how often is it right?

Recall asks:

> Of all customers who actually churned, how many did the model catch?

Precision matters when false positives are expensive.

Recall matters when false negatives are expensive.

---

# ROC-AUC

ROC-AUC measures how well the model separates positive and negative cases across thresholds.

It is useful for understanding ranking ability, but it can look overly optimistic when the positive class is rare.

---

# Precision-Recall Curves

Precision-Recall curves focus directly on the positive class.

For imbalanced datasets like churn, fraud, and medical diagnosis, PR curves often tell a more practical story than ROC curves.

---

# Threshold Tuning

The default threshold of `0.5` is not sacred.

Changing the threshold changes the model's behavior:

- lower threshold: more positives, higher recall, more false positives
- higher threshold: fewer positives, higher precision, more false negatives

Threshold tuning turns evaluation into a business decision.
